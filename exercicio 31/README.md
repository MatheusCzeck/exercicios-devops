"# Testes De Software" 
**1. Conceitue os principais níveis de testes estudados ao longo da disciplina (Testes de Unidade, Testes de Integração, Testes de Sistema e Testes de Aceitação).**

Os testes de software são divididos em diferentes níveis, cada um com uma finalidade específica dentro do processo de garantia da qualidade.

O **Teste de Unidade** é considerado o nível mais básico de teste. Ele é utilizado para verificar se pequenas partes do sistema, como funções, métodos ou classes, estão funcionando corretamente de forma isolada. Esse tipo de teste ajuda a identificar erros logo no início do desenvolvimento, tornando as correções mais simples e menos custosas.

O **Teste de Integração** é realizado após os testes de unidade e tem como objetivo verificar se os diferentes componentes do sistema conseguem se comunicar corretamente. Nessa etapa, são avaliadas integrações com bancos de dados, APIs e outros serviços, garantindo que a troca de informações aconteça sem problemas.

Já o **Teste de Sistema** avalia a aplicação como um todo. O foco é verificar se todas as funcionalidades trabalham juntas de acordo com os requisitos definidos. Nesse nível, o sistema é analisado sob a perspectiva do usuário, considerando o comportamento completo da aplicação.

Por fim, o **Teste de Aceitação** é a etapa em que o software é validado pelo cliente ou pelos usuários finais. Seu principal objetivo é confirmar que o sistema atende às necessidades do negócio e está pronto para ser utilizado em ambiente de produção.

**2. Descreva quais os insumos de entrada necessários para o planejamento e a modelagem eficaz das suítes de teste.**

Para que os testes sejam bem planejados, é fundamental contar com informações confiáveis sobre o sistema que será avaliado. Os principais insumos utilizados são os documentos de requisitos funcionais e não funcionais, histórias de usuário, regras de negócio, diagramas de arquitetura, modelos de banco de dados e documentação das APIs.

Esses materiais servem como base para que a equipe compreenda o funcionamento esperado da aplicação e consiga criar cenários de teste adequados. Quando os requisitos estão bem definidos, torna-se mais fácil identificar possíveis riscos e garantir uma cobertura mais completa das funcionalidades do sistema.

**3. Descreva as principais atividades do ciclo de vida de testes (planejamento, execução, reporte de bugs), como a automação pode ser aplicada nessas etapas e qual a importância de uma boa cobertura de testes para a resiliência do produto final.**

O ciclo de vida dos testes começa com o **planejamento**, etapa em que são definidos os objetivos dos testes, quais funcionalidades serão avaliadas, os recursos necessários e os critérios que determinarão o sucesso da validação. Um bom planejamento é essencial para direcionar os esforços da equipe e evitar retrabalho.

Em seguida, ocorre a **execução dos testes**, quando os cenários planejados são colocados em prática para verificar se o sistema apresenta os resultados esperados. Durante essa fase, podem ser identificados defeitos, comportamentos inesperados ou falhas de integração.

Quando um problema é encontrado, entra em cena o **reporte de bugs**, que consiste em registrar detalhadamente a falha para que a equipe de desenvolvimento consiga reproduzi-la e corrigi-la. Um relatório bem elaborado facilita a resolução do problema e melhora a comunicação entre as equipes.

A **automação de testes** pode contribuir significativamente em todo esse processo, principalmente na execução de testes repetitivos e de regressão. Com ferramentas automatizadas, é possível executar diversos cenários em menos tempo, com maior precisão e frequência, aumentando a eficiência da equipe e reduzindo a possibilidade de erros humanos.

Além disso, uma boa **cobertura de testes** é fundamental para a qualidade do produto final. Quanto mais funcionalidades e cenários forem testados, menor será o risco de falhas chegarem ao usuário final. Isso torna o sistema mais confiável, facilita futuras manutenções e contribui para a resiliência da aplicação, garantindo que ela continue funcionando corretamente mesmo diante de mudanças e atualizações.
